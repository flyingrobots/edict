## Release Thesis

`v0.8.0-alpha.1` proves the first honest effectful compiler path: one minimal
annotated effectful `let ... else` source shape lowers through the compiler
spine into stable typed Core, using the file-backed authority facts introduced
in `v0.7.0-alpha.1`, while unsupported effectful forms fail with stable
compiler error kinds before Core lowering.

The release deliberately excludes target IR generation, target-runtime
execution, adapter composition, public CLI workflows, admission execution,
trusted lawpack or target-profile authorship governance, broad source-language
lowering, and crates.io publication.

## Included

- Public Core model support for `CoreNode::Effect` and
  `CoreObstructionArm`.
- Reference canonical encoding for semantic Core effect nodes, including
  mutation-sensitivity coverage for effect coordinates.
- A minimal annotated effectful source shape:
  `let name: Type = effect(arg) else { failure(binder) => Obstruction };`.
- Compiler-spine lowering from that source shape into typed Core using
  file-backed profile, budget, profile write-class, and effect write-class
  facts.
- Deterministic obstruction map lowering with binder identities stabilized by
  failure key rather than source arm order.
- Stable rejection of unsupported effectful branch-yield lowering, chained
  effect calls, typed effect calls, and duplicate obstruction failure keys
  before Core lowering.
- Core golden discipline for the existing pure `bounded-hello` reviewed bytes
  and digest; the pure Core golden fixture remains stable for this release.
- Compiler-spine and Core IR topic-shelf updates describing the landed
  effectful Core boundary.
- Post-`v0.7.0-alpha.1` release-process evidence updates that mark the
  file-backed authority-facts alpha as published without changing the immutable
  release tag.

## Explicit Non-Goals

The `v0.8.0-alpha.1` release explicitly does not include:

- crates.io publication;
- target IR generation;
- target-runtime execution;
- adapter composition;
- public CLI workflows;
- admission execution workflow;
- trusted lawpack or target-profile authorship governance implementation;
- broad source-language lowering;
- bare effect-statement lowering;
- effectful branch-yield lowering;
- obstruction payload lowering;
- target or lawpack failure-fact exhaustiveness;
- full lawpack or target-profile manifest loading.
