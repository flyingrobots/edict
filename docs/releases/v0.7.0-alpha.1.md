## Release Thesis

`v0.7.0-alpha.1` advances Edict's compiler authority boundary by moving the
first compiler context facts from caller-constructed memory fixtures into
explicit, digest-bound authority-facts files. It gives maintainers executable
evidence that operation profiles, profile write-class allowances, effect write
classes, and budgets can be loaded deterministically from `lawpack` and
`targetProfile` source identities.

The release deliberately excludes trusted authorship governance, full
lawpack/target-profile manifest loading, obstruction/obligation/adapter/
footprint/cost/target-capability corpus loading, target IR generation, full
effectful source lowering, admission execution, and crates.io publication.

- `xtask` remains an internal workspace helper at `0.0.0`.
- Versions currently track repository/specification maturity, not crates.io API
  stability.

## Included

- File-backed authority-facts loading in `edict_syntax` through
  `load_authority_facts_file`,
  `load_compiler_context_from_authority_fact_files`, and
  `compiler_context_from_authority_facts`.
- Digest-bound `lawpack` and `targetProfile` authority source identities.
- Operation-profile facts, profile write-class allowances, effect write-class
  facts, and budget facts for the first compiler context fact set.
- Compiler-spine harness coverage proving loaded facts can compile
  `bounded-hello` and can reject a read-only profile/write-effect mismatch with
  `ProfileEffectMismatch`.
- Stable `AuthorityFactsLoadFailureKind` categories for malformed JSON,
  unreadable paths, invalid API versions, unsupported source kinds, missing or
  invalid coordinates, non-digest-locked sources, invalid write classes, and
  conflicting facts.
- The authority-facts topic shelf plus lawpack, target-profile, and
  compiler-spine test-plan links to the file-backed fact path.
- `docs/design/authority-fact-governance.md`, a non-topic design note for the
  trusted lawpack and target-profile authorship questions opened by file-backed
  fact loading.
- Release-process and Rust-standards policy hardening, including release thesis,
  previous-tag diff reconciliation, durable release reports, no-crates evidence,
  a deny-level missing-`Debug` lint ratchet, and a review-bot fallback policy.

## Explicit Non-Goals

The `v0.7.0-alpha.1` release explicitly does not include:

- crates.io publication;
- trusted lawpack or target-profile authorship workflow;
- full `edict.lawpack/v1` manifest loading;
- full `edict.target-profile/v1` manifest loading;
- obstruction, obligation, adapter, footprint, cost, or target-capability
  corpus loading;
- global registry, trust root, identity system, or revocation model;
- target IR generation;
- full effectful source lowering;
- admission execution workflow;
- Continuum participant trust policy.
