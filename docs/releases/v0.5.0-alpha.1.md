# v0.5.0-alpha.1 Release Notes

Target date: 2026-08-12

Status: published as a GitHub prerelease from the `v0.5.0-alpha.1` tag
targeting `e680adf0a3ccc72559d1c4710e81010738918532`.

## Release Type

`v0.5.0-alpha.1` is the bundle and admission alpha. It adds typed Gate C
admission-boundary checks for the Edict-owned side of Continuum participation:
admission requests, admission receipts, policy epochs, and invocation capability
evidence.

This is not a Continuum participant-policy engine, admission ledger, identity,
delegation, or revocation release.

## Version Policy

- `crates/edict-syntax` is versioned as `0.5.0-alpha.1`; its package remains
  `publish = false`.
- `xtask` remains an internal workspace helper at `0.0.0`.
- Versions currently track repository/specification maturity, not crates.io API
  stability.

## Included

- Typed `AdmissionRequest`, `AdmissionReceiptBody`, `AdmissionCapability`, and
  `GateCInvocation` contracts in `edict_syntax`.
- Domain-separated `digest_admission_request` support for binding receipts to
  the exact admission request they answer.
- `validate_admission_request` checks for bundle-subject membership,
  operation-requirement binding, hidden execution input rejection, and
  required-operation evidence.
- `validate_admission_receipt` checks for receipt echoing, request-digest
  echoing, receipt acyclicity, accepted/rejected evidence consistency,
  policy-epoch binding, and admitted operations/capabilities staying within the
  original request.
- `check_gate_c_invocation` checks for bundle-subject alignment, admitted
  operation membership, participant-matched invocation capability evidence, and
  invocation capability scope admitted by the receipt.
- Admission topic shelf updates for the Edict-owned Gate C contract and
  verification matrix.
- Release-process automation for successful `release/*` prep merges: main CI
  creates an immutable release tag, dispatches publication, and closes the
  matching milestone when the release exists and the milestone has zero open
  issues.

## Explicit Non-Goals

The `v0.5.0-alpha.1` release explicitly does not include:

- crates.io publication;
- Continuum participant policy evaluation;
- participant identity, delegation, revocation, or trust graph evaluation;
- admission ledger persistence or transparency-log publication;
- signature verification;
- file-backed admission request or receipt loading;
- target lowerer execution;
- Target IR generation from Core;
- contract-bundle semantic or release digest recomputation from canonical
  preimages;
- a compiler, lowerability, bundle, or admission CLI.

## Required Verification

Before the release-prep PR can merge:

- `cargo test -p xtask release_` must pass.
- `cargo xtask verify` must pass on the release-prep branch.
- GitHub Actions on the release-prep PR must pass for MSRV Rust 1.85.0 and
  stable Rust.
- Issues #6, #11, and #42 must be closed or explicitly deferred.
- The `v0.5.0-alpha.1` milestone must have no open release-blocking issues when
  publication runs.
- Release notes must distinguish Edict-owned admission artifacts from
  Continuum-owned participant policy, identity, delegation, and revocation.
