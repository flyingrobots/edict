## Release Type

`v0.11.0-alpha.1` is the contract-bundle assembly and canonical Target IR
artifact freeze alpha. It packages two cryptographic boundaries:

- participant-neutral contract-bundle assembly with reviewed semantic/release
  bundle digest goldens;
- canonical Target IR artifact bytes and digest goldens for the current Echo
  and git-warp review artifacts.

It does not add runtime execution, admission execution, verifier completeness,
general target plugin dispatch, canonical `ContractBundleManifest` bytes, or

## Release Thesis

`v0.11.0-alpha.1` proves that Edict can assemble a participant-neutral contract
bundle manifest from a real Core module and digest-locked artifact graph, compute
the top-level semantic and release bundle digests from the reviewed v0.11
preimage shapes, and validate the assembled manifest. It also freezes canonical
Target IR artifact bytes and `edict.target-ir.artifact/v1` digests for the
current Echo and git-warp review artifacts, then wires that computed Target IR
digest into bundle assembly without allowing two independent target IR digest
sources.

The release deliberately narrows its claim to deterministic bytes, digests,
goldens, validation, and assembly for the supported alpha shapes. It does not
claim target runtime execution, Echo verifier completeness, git-warp commit
creation, git-warp CRDT reducer verification, participant admission policy,
general target plugin dispatch, or canonical bytes for the full bundle manifest.

## Included

- `assemble_contract_bundle(...)`, which computes the Core digest from a real
  `CoreModule`, consumes typed digest-locked references, computes
  `semanticBundleDigest` and `releaseBundleDigest`, fills a
  `ContractBundleManifest`, rejects invalid assembly inputs, and validates the
  manifest before returning it.
- Exact semantic and release bundle digest preimage ordering for the v0.11
  assembly boundary, including structured release provenance for source
  descriptors and tool identities.
- Bundle digest mutation coverage that proves semantic-layer changes move both
  top-level bundle digests, while release-only provenance and toolchain changes
  move only `releaseBundleDigest`.
- Reviewed bundle digest goldens under `fixtures/bundle/assembly/`, regenerated
  by `cargo xtask bundle-goldens --write` and checked by
  `cargo xtask bundle-goldens --check`.
- Canonical Target IR artifact value model and canonical CBOR encoding for the
  current common `TargetIrArtifact` envelope.
- `edict.target-ir.artifact/v1` digest framing, with the artifact domain
  retained inside the canonical value.
- Reviewed Echo and git-warp Target IR byte/digest goldens under
  `fixtures/target-ir/canonical/`, regenerated by
  `cargo xtask target-ir-goldens --write` and checked by
  `cargo xtask target-ir-goldens --check`.
- Target IR determinism and mutation coverage for map construction order, step
  order, target-profile digest, source Core coordinate, intent name, effect
  coordinate, selected intrinsic, input expression, obstruction failures and
  arms, input constraints, budget, and result expression.
- `assemble_contract_bundle_from_target_ir(...)`, which computes
  `targetIrDigest` from a real `TargetIrArtifact`, derives the target-profile
  reference from that artifact, rejects Core/Target IR source-coordinate
  mismatches, and reports invalid embedded target-profile digests with a stable
  field.
- `cargo xtask verify` now checks Core, Target IR, and bundle goldens.
- Release-prep issue #109 and milestone #12 evidence for the v0.11 release
  boundary.

## Explicit Non-Goals

The `v0.11.0-alpha.1` release explicitly does not include:

- crates.io publication;
- target runtime execution;
- participant admission execution;
- participant policy evaluation, identity, delegation, or revocation logic;
- Echo verifier completeness;
- git-warp commit object creation;
- git-warp CRDT reducer verification;
- general `edict-target-lowerer.wit` plugin dispatch;
- additional target profiles beyond the current Echo and git-warp review
  artifacts;
- source-to-target fixtures beyond the currently supported compiler spine;
- canonical `ContractBundleManifest` bytes.
