# v0.6.0-alpha.1 Release Notes

Target date: 2026-08-26

Status: published as a GitHub prerelease from the `v0.6.0-alpha.1` tag
targeting `ec1ce8a5186801caa3c627c6714cedff7fdf43bc`.

## Release Type

`v0.6.0-alpha.1` is the developer-tooling alpha. It adds editor-facing lexical
highlighting roles, first-party grammar artifacts, and a thin VS Code/Cursor
package for `.edict` syntax highlighting.

This is not a compiler, language-server, marketplace-publication, target
lowering, or admission-tooling release.

## Version Policy

- `crates/edict-syntax` is versioned as `0.6.0-alpha.1`; its package remains
  `publish = false`.
- `xtask` remains an internal workspace helper at `0.0.0`.
- Versions currently track repository/specification maturity, not crates.io API
  stability.

## Included

- Editor-facing lexical highlighting in `edict_syntax`: `highlight_source`,
  `HighlightToken`, and stable `HighlightRole` values for comments,
  identifiers, keywords, numbers, operators, punctuation, strings, and type
  identifiers.
- A deterministic developer-tooling fixture for highlight role coverage.
- Tree-sitter grammar source, generated parser source, node metadata, highlight
  query, and current-subset corpus aligned with the reference parser.
- TextMate grammar artifact for `.edict` lexical scopes aligned with the public
  highlighter roles.
- Thin VS Code/Cursor extension package under `editors/vscode/` that registers
  `.edict`, contributes comment and bracket metadata, and uses the canonical
  TextMate grammar.
- Developer-tooling topic shelf updates covering highlight roles, Tree-sitter,
  TextMate, and VS Code/Cursor behavior.
- Fixture, lawpack, and assurance topic shelves for cross-cutting current-truth
  contract surfaces.
- Release-prep topic-shelf audit gate requiring `docs/topics/` coverage and
  accuracy to both meet at least 90%.

## Explicit Non-Goals

The `v0.6.0-alpha.1` release explicitly does not include:

- crates.io publication;
- a compiler CLI;
- a language server;
- editor diagnostics, semantic tokens, or formatter support;
- VS Code Marketplace or Open VSX publication;
- Vim, Zed, or jedit extension packages;
- generated Tree-sitter package publication;
- Tree-sitter runtime integration in the VS Code/Cursor package;
- source-language lowering beyond the initial pure local-record subset;
- target lowerer execution or Target IR generation;
- admission execution tooling;
- executable lawpack, HOLMES, Watson, or Moriarty tooling.
